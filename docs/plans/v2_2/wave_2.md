# wave 2: biome linter

add biome as the project linter and formatter. configure it, fix all lint errors, add a `lint` script to every package and the root. after this wave, every PR gets static analysis beyond TypeScript's type checker.

## why biome

- **fast:** written in Rust, 10-100x faster than eslint for large codebases
- **zero-config for TS/TSX:** works out of the box with TypeScript and React
- **replaces eslint + prettier:** one tool for linting AND formatting
- **growing ecosystem:** biome is the successor to Rome; actively maintained

### 2.4 remove stale eslint references

the codebase has `// eslint-disable-next-line` comments but no eslint config. these are dead comments.

**files:** grep for `eslint-disable` across all `.ts`/`.tsx` files and remove the comments (biome uses different directives: `// biome-ignore`).

**mirror:** no in-repo precedent for biome-style suppressions yet. The biome equivalent syntax is `// biome-ignore lint/suspicious/noExplicitAny: <reason>`. Mirror eslint's "explain why" convention by adding a short reason after the colon — verified against biome docs at https://biomejs.dev/linter/#suppress-lint-rules-with-comments.

**context — exact stale comment locations (verified by `grep -rn "eslint-disable" packages/ --include="*.ts" --include="*.tsx"`):**
```
packages/core/tests/server/auth-middleware.test.ts:34:    // eslint-disable-next-line @typescript-eslint/no-explicit-any
packages/core/src/tools/linear/issues.ts:31:// eslint-disable-next-line @typescript-eslint/no-explicit-any
packages/core/src/tools/linear/issues.ts:116:    // eslint-disable-next-line @typescript-eslint/no-explicit-any
packages/core/src/server/middleware/auth.ts:68:  // eslint-disable-next-line @typescript-eslint/no-explicit-any
packages/core/src/persistence/factory.ts:43:    // eslint-disable-next-line @typescript-eslint/ban-ts-comment
packages/cli/src/commands/console.ts:7:  // eslint-disable-next-line no-console
packages/cli/src/commands/console.ts:9:  // eslint-disable-next-line no-console
```
For each line: (a) if the underlying issue (the `any` cast, the `console.log`, the `@ts-expect-error`) is still present AND biome flags it, replace with `// biome-ignore lint/<rule>: <reason>` — for example `noExplicitAny` becomes `// biome-ignore lint/suspicious/noExplicitAny: better-auth options bag is untyped`; `no-console` becomes `// biome-ignore lint/suspicious/noConsole: cli output channel`. (b) If biome does NOT flag the underlying issue (because the rule is off in `recommended` or the auto-fix in 2.2 already resolved it), just delete the comment outright.

**conventions:** ESM `.ts`/`.tsx` files only; the comments live above the line they suppress (eslint's "next-line" semantics) — biome's `// biome-ignore` follows the same "next-line" convention. Always include a reason after the colon — `// biome-ignore lint/<rule>:` (no reason) is itself flagged. Run `bun run lint` after each batch of edits to confirm the rewrite suppresses the right rule.

**acceptance:**
- [x] `grep -r "eslint-disable" packages/ --include="*.ts" --include="*.tsx"` returns nothing (or only in `node_modules`)
