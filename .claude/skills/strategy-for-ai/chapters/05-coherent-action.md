# Ch. 5 — Coherent action

Sequenced moves that compound.

The third leg of the kernel is the set of actions that advance the guiding policy. The word that matters most is **coherent.** Not a list. Not parallel initiatives. A set of moves that *fit together* — each one strengthening the conditions for the next.

Rumelt's metaphor: actions in a real strategy are like the moves of a chess player. Each one prepares the next. The third move makes the fifth move possible. Without that interlocking, you have *activity,* not strategy.

A coherent set of actions has three qualities:

### Sequence

Order matters. Action B depends on Action A being underway. Action C only makes sense after B has propagated. If you could shuffle the order with no loss, the actions aren't coherent — they're just things you wanted to do.

The test: draw the actions as a chain, with arrows showing what each one *enables.* If the diagram is just a bulleted list, the strategy isn't done.

### Concentration

Most resources go to the move with the most leverage. You don't divide effort evenly across actions; you weight them according to their role in the sequence. The early-sequence actions get the most attention; the later ones get progressively less but rest on what came before.

This is where most "strategies" collapse — every action gets a roughly equal share of resources because every team owns one. Equal weight = no strategy. (More on this in Ch. 7.)

### Resonance

The actions reinforce each other. The marketing action makes the product action more effective. The product action makes the pricing action work. They're not parallel tracks; they're a chord.

The opposite is "actions" that work in isolation. If you stripped any single one out, the rest still function unchanged. That's a portfolio, not a strategy.

## Common failure modes

**The feature list.** "Strategy actions" that are actually a backlog. Each item is independent. Order doesn't matter. Each team owns one and pursues it autonomously. This is what happens when "alignment" is prioritized over coherence — everyone got their pet project on the list.

**The big-bang launch.** All actions assumed to land at the same time. No sequencing because everything is "phase 1." Usually because admitting some things are phase 2 would require admitting they're not getting done this quarter.

**The orphaned actions.** Actions whose enabling move is missing. *"We'll launch in Europe"* without *"we'll first build the localization infrastructure."* The action exists in the plan; the precondition doesn't.

## What good looks like

- Three to five actions, sequenced, each enabling the next
- Most resources concentrated on early-sequence actions
- Actions whose effects compound — A makes B possible makes C unstoppable
- An explicit dependency map between actions

## What bad looks like

- Six to twelve "priorities," each owned by a different team, in parallel
- Even resource distribution across actions
- Actions interchangeable in order
- "Phase 1: everything" with no real sequencing

## Diagnostic questions

- What's the sequence? Which action comes first, and why?
- Where is most of our resource concentrated? Does it match the leverage point in the sequence?
- How does each action reinforce the next?
- If we re-ordered the actions, would anything be lost?

## Craft move

Draw the actions as a chain, with arrows that name what the previous unlocks. If the diagram is just bullets, the strategy isn't done — you have a list of intentions, not a coherent plan.
